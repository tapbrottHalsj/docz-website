# Components API

> ### Note
>
> By default this components just works if is inside a docz theme component, because it need a lot of data and a be in a docz environment!

## `<Docs>`

This is a [render props](https://reactjs.org/docs/render-props.html) component that you can use inside your theme to get the list of all docs parsed. It can be usefull when you want a mount something like a menu or a list.

```js
import { Docs, Link } from 'docz'

const App = () => (
  <Docs>
    {({ docs }) => (
      <MyMenu>
        {docs.map(doc => (
          <Link key={doc.id} to={doc.route}>
            {doc.name}
          </Link>
        ))}
      </MyMenu>
    )}
  </Docs>
)
```

### Properties

`none`

### Children props

- docs: `Docs[]`<br />
  All documents parsed by docz

- menus: `string[]`<br />
  An array of unique menus setted on your docs

### Typings Definitions

```ts
interface Docs {
  id: string
  filepath: string
  slug: string
  route: string
  name: string
  menu: string | null
  order: number
  settings: {
    [key: string]: any
  }
}

interface RenderProps {
  docs: Docs[]
  menus: string[]
}

interface DocsProps {
  children: (props: RenderProps) => React.ReactNode
}

export const Docs: React.SFC<DocPreviewProps>
```

## `<DocPreview>`

This is the main component when you want to create your theme. It is responsible to manage all routes based on your parsed documents, make automatic code splitting and receive your components map to render. Use it like that:

```js
import React from 'react'
import { DocPreview, theme } from 'docz'

import MyCoolSidebar from './my-cool-sidebar'
import * as components from './my-component-list'

const Theme = theme(() => (
  <div>
    <MyCoolSidebar />
    <DocPreview
      components={{
        page: components.Page,
        h1: components.H1,
        h2: components.H2,
        pre: components.Pre,
      }}
    />
  </div>
))
```

### Properties

- **components:** `ComponentsMap`<br />
  This map is responsible to render this components inside your `.mdx` compiled file.

### Type Definitions

```ts
interface ComponentsMap {
  loading?: React.ComponentType
  page?: React.ComponentType<PageProps>
  notFound?: React.ComponentType<RouteComponentProps<any>>
  render?: RenderComponent
  h1?: React.ComponentType
  h2?: React.ComponentType
  h3?: React.ComponentType
  h4?: React.ComponentType
  h5?: React.ComponentType
  h6?: React.ComponentType
  ul?: React.ComponentType
  table?: React.ComponentType
  pre?: React.ComponentType
  inlineCode?: React.ComponentType
}

interface DocPreviewProps = {
  components: ComponentsMap
}

export const DocPreview: React.SFC<DocPreviewProps>
```

## `<Link>`

Use this component to create links inside your theme.

```js
import React from 'react'
import { Link } from 'docz'

const App = () => <Link to="/my-document-route">Name</Link>
```

### Properties

- to: `string`<br />
  The path that you want to visit

- activeClassName: `string`<br />
  Class that will be used when your link is active

- activeStyle: `string`<br />
  Style that will be used when your link is active

- isActive: `(match, location) => boolean`<br />
  Function to define the active logic of your Link

### Type Definitions

```ts
import { NavLinkProps } from 'react-router-dom'

export const Link: React.SFC<NavLinkProps>
```

## `<Playground>`

Component used to render your component inside a playground and showing the code used for that.

![](https://cdn-std.dprcdn.net/files/acc_649651/bG0ZfH)

```markdown
---
name: MyComponent
---

import { Playground } from 'docz'
import { MyComponent } from './MyComponent''

# Some title

<Playground>
  <MyComponent />
</Playground>

<Playground>
  {() => (
    <MyComponent />
  )}
</Playground>
```

> ### Note
>
> If you want to create some variables inside your playground you can render the children as a function.

### Properties

`none`

### Type Definitions

```ts
interface PlaygroundProps {
  children: React.ReactNode | () => React.ReactNode
}
```

## `<PropsTable>`

Component that take a component and generate a table of properties based on your properties definitions.

![](https://cdn-std.dprcdn.net/files/acc_649651/ly7vvI)

```markdown
---
name: MyComponent
---

import { PropsTable } from 'docz'
import MyComponent from './MyComponent'

<PropsTable of={MyComponent} />
```

> ### Important
>
> To get right with `<PropsTable>` you need to export your component as a `default` export, this is a very bad limitation of React Docgen, but is something necessary right now!

### Properties

- **of:** `React.ComponentType<any>`

## `<ThemeConfig>`

Use this component to retrieve the `themeConfig` object passed on your project configuration. By default Docz merge your `themeConfig` object with the default theme config of each theme.

```js
import React from 'react'
import { ThemeConfig } from 'docz'

const MyDeepComponentInsideMyTheme = () => (
  <ThemeConfig>{config => <div>{config.title}</div>}</ThemeConfig>
)
```

### Properties

`none`

### Children Props

- **config:** `ThemeConfig`

### Type Definitions

```ts
interface ThemeConfig {
  title: string
  description: string
  [key: string]: any
}

interface RenderProps {
  config: ThemeConfig
}

interface ThemeConfigProps {
  children: (props: RenderProps) => React.ReactNode
}

export const ThemeConfig: React.SFC<ThemeConfigProps>
```

## `theme()`

If you want to create your custom theme you need to use `theme()` function, that is a High Order Component that receive as the first parameter a `config` property and return a function that receive your theme component as parameter:

```js
import React from 'react'
import { theme, DocPreview } from 'docz'

const Theme = () => (
  <div>
    <MyCoolestSidebar />
    <DocPreview />
  </div>
)

const themeConfig = {
  /* ... */
}

export default theme(themeConfig)(Theme)
```

### Type Definitions

```ts
import { ComponentType, ReactNode } from 'react'

interface ThemeConfig {
  [key: string]: any
}

function theme(config: ThemeConfig): (Component: ComponentType) => ReactNode
```
